# 水资源知识图谱扩展指南 (Knowledge Graph Expansion Guide)

为了让你的大模型和问答系统发挥最大价值，建议按照 **“实体化 -> 关联化 -> 动态化”** 的路径来扩充知识图谱。

以下是优先级最高的 4 类数据扩充方向，以及它们能解锁的新问答场景：

## 1. 将“属性”升级为“节点”（最高性价比）
当前你的 `城市`、`省份`、`流域` 只是水库节点上的字符串属性。若将它们抽取为独立节点，图谱的推理能力将成倍提升。

*   **新增节点**：`流域 (Basin)`、`省份 (Province)`、`城市 (City)`
*   **新增关系**：`(水库)-[LOCATED_IN]->(城市)`、`(水库)-[BELONGS_TO]->(流域)`
*   **解锁问答**：
    *   *“长江流域包含哪些防洪库容超过100亿立方米的水库？”*
    *   *“湖北省有哪些大型水库？”*
*   **Cypher 改造示例**：
    ```cypher
    // 自动将图谱中现有的流域属性抽取为独立节点
    MATCH (r:水库)
    WHERE r.river_basin IS NOT NULL
    MERGE (b:流域 {name: r.river_basin})
    MERGE (r)-[:BELONGS_TO_BASIN]->(b);
    ```

## 2. 补充空间与水系拓扑数据（水利核心领域）
水库不是孤立的，它们是水系网络上的节点。引入河流和上下游测站能极大增强图谱的空间感知。

*   **新增节点**：`河流 (River)`、`水文站 (HydrologicalStation)`
*   **新增关系**：`(水库)-[LOCATED_ON]->(河流)`、`(河流)-[FLOWS_INTO]->(河流)`、`(水文站)-[MONITORS]->(河流/水库)`
*   **解锁问答**：
    *   *“汉江上有哪些水利枢纽？”*
    *   *“丹江口水库下游有哪些水文站？”*（这对于防洪应急问答非常有价值）

## 3. 扩充工程与管理实体（政务/管理场景）
如果你的系统面向水利管理人员，工程背景和管理归属最为关键。

*   **新增节点**：`水力发电站 (HydropowerStation)`、`管理单位 (ManagementEntity)`、`建设单位 (ConstructionEntity)`
*   **新增关系**：`(水电站)-[ATTACHED_TO]->(水库)`、`(管理单位)-[MANAGES]->(水库)`
*   **解锁问答**：
    *   *“负责管理三峡水库的机构全称是什么？”*
    *   *“长江水利委员会管辖了哪些大型水库？”*

## 4. 引入时序/动态数据（高阶场景）
目前的图谱是静态的（死水位、正常蓄水位），如果接入实时或历史特征，系统将具备真正的业务生命力。

*   **新增节点**：`水位记录 (WaterLevelRecord)`、`洪旱灾害事件 (DisasterEvent)`
*   **新增关系**：`(水库)-[HAS_RECORD]->(水位记录)`、`(水库)-[AFFECTED_BY]->(灾害事件)`
*   **解锁问答**：
    *   *“这可以通过大模型配合 Function Calling，查询：小浪底水库当前水位是否超过了汛限水位？”*
